# UCSD Historical Enrollment Data
A repository that hosts some basic historical enrollment data for Spring 2022 and later.

## Content
This repository contains CSV files which contains information regarding the number of seats available at some time for certain classes.

I plan on updating this repository somewhat frequently during enrollment periods. 

## Source & Information
The raw CSV files were generated by my [WebReg Wrapper](https://github.com/ewang2002/ucsd_webreg_rs), specifically the tracker and the associated local web server.

### General Notes
- The raw files may have some corrupted rows (especially the last rows). This is because I am using a buffered writer, which only occasionally writes to the file. Sometimes, I end up terminating the application before it can write to the file.
- There may be some gaps in time. For example, there may be time periods of 15+ minutes with no data. 
- Data in general may not be collected at consistent intervals. For example, the time between two requests to some class X may vary (e.g. 8 minutes vs. 8 minutes and 30 seconds). 
- Additionally, because WebReg does maintenance at random times, there may be long periods of time with no data. 

### Data Collection Notes
Spring 2022 was the first quarter where I actually started tracking course enrollment counts for certain classes. So, it should come as no surprise that I made a fair number of mistakes in this quarter that I hopefully won't make again in future quarters.

In Spring 2022, the one major mistake I made was collecting the number of available seats (e.g. 17/35 means 17 seats are left) instead of the number of students enrolled (e.g. 17/35 means 17 students are enrolled) for each course. Due to various factors (e.g. some sections may have more students than the limit), collecting the number of students enrolled is significantly more useful than collecting the number of available seats.

In `SP22D`, the summer sessions, and future quarters, I will be collecting the number of students enrolled instead.

### Data Cleaning
Cleaning up was done by the few (poorly written) Python scripts that are in this repository. So, you're probably interested in the following folders:
- `cleaned`: A folder containing one CSV file that consists of all CSV files merged together from `raw`. Depending on the size of this file, it might be omitted; if it is omitted, you can recreate it by using the `run.sh` script.
- `overall`: A folder containing one CSV file per *course* (e.g. CSE 8A with 2 sections would both be put in a file called `CSE 8A.csv`). 
- `section`: A folder containing one CSV file per *section* (e.g. CSE 8A with 2 sections `A` and `B` would be put in two files called `CSE 8A_A.csv` and `CSE 8A_B.csv`).

## Plotting
There is a Python script, `plot.py`, which is responsible for the graphs that are in this repository. `plot.py` depends on a configuration file `plotconfig.py`, which should be located in the root of each directory containing the data of interest.

To see an example of what this may look like, check out `plotconfig_example.py` or `plotconfig.py` in one of the folders.

A repository containing enrollment data for Spring 2022, Summer 2022, Fall 2022, and possibly more quarters.

## Repository Content

This repository contains CSV files which contains information regarding the number of seats available at some time for certain classes.

Please view each folder's README to learn more about what information was tracked.

## Source & Process
The raw CSV files were generated by my [WebReg Wrapper](https://github.com/ewang2002/ucsd_webreg_rs).

### Data Collection Information
<details>
<summary>Click Here.</summary>
<br>

The way I collected data certainly wasn't great. In particular, several issues to expect are
- the last row of each raw CSV file may be corrupted, and should be removed; `clean_raw_csvs.py` should take care of this, and 
- timestamps of each row may not be consistent (e.g., inconsistent intervals between requests), or there may be periods of time with no data (e.g., hours of no data). 

In other words, the method of data collection is not perfect. However, for most purposes (e.g., data visualization), it should suffice.
</details>



### Data Collection Notes
<details>
<summary>Click Here.</summary>
<br>

Spring 2022 was the first quarter where I actually started tracking course enrollment counts for certain classes. So, it should come as no surprise that I made a fair number of mistakes in this quarter that I hopefully won't make again in future quarters.

In Spring 2022, the one major mistake I made was only collecting the number of available seats (e.g. 17/35 means 17 seats are left) instead of the number of students enrolled (e.g. 17/35 means 17 students are enrolled) for each course. Due to various factors (e.g., some sections may have more students than the limit), collecting the number of students enrolled is significantly more useful than collecting the number of available seats.

In `SP22D`, the summer sessions, and future quarters, I will be collecting the number of students enrolled instead.

Additionally, note that Spring 2022 and Summer Session I/II 2022 only has enrollment data for all lower- and upper-division COGS, CSE, MATH, and ECE courses. Starting Fall 2022, all lower- and upper- division courses will be tracked; only COGS, CSE, MATH, and ECE graduate courses will be tracked.

</details>

### Data Cleaning
<details>
<summary>Click Here.</summary>
<br>

Cleaning up was done by the few Python scripts that are in this repository. So, you're probably interested in the following folders:
- `cleaned`: A folder containing one CSV file that consists of all CSV files merged together from `raw`. Depending on the size of this file, it might be omitted; if it is omitted, you can recreate it by using the `run.sh` script.
- `overall`: A folder containing one CSV file per *course* (e.g. CSE 8A with 2 sections would both be put in a file called `CSE 8A.csv`). 
- `section`: A folder containing one CSV file per *section* (e.g. CSE 8A with 2 sections `A` and `B` would be put in two files called `CSE 8A_A.csv` and `CSE 8A_B.csv`).
</details>
